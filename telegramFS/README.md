# Telegram File Interface
Интерфейс к мессенджеру, построенному на архитектуре "User Space Proxy" в связке с Linux Модулем ядра.

## Описание архитектуры
Система состоит из двух частей строго по ТЗ:
1. **Модуль ядра (`telegram_fs.ko`)**: отвечает за предоставление файлового интерфейса пользователю. Он создаёт директорию `/proc/telegram/` и 3 ноды чатов: `chat_1`, `chat_2`, `chat_3`. Также он создаёт ноду `ctrl`. 
При каждом чтении или записи в чат, модуль ядра перехватывает операцию, блокирует клиентский процесс (`cat`/`echo`), и передает запрос на сервер через контрольный файл.
2. **Пользовательская программа (`telegram_server`)**: хранит базу данных чатов (историю сообщений). Она висит в цикле чтения файла `/proc/telegram/ctrl`. 
Как только приходит запрос на чтение от модуля, сервер форматирует историю в строку и отдаёт обратно через контрольный файл. Модуль подхватывает строку и отдаёт клиенту.

Это обеспечивает гибкую логику — ядро просто работает почтальоном, а вся память и логика чатов вынесена в пользовательское пространство (User Space).

## Инструкция по сборке
```bash
# 1. Компиляция модуля и сервера (должен быть подписан если включен SecureBoot)
make
```

## Инструкция по запуску
```bash
# 1. Загрузка модуля (создаст /proc/telegram/*)
sudo insmod telegram_fs.ko

# 2. Запуск сервера в фоне
./telegram_server &

# 3. Работа с чатами - Чтение
cat /proc/telegram/chat_1
В чате пока нет сообщений.

# 4. Работа с чатами - Запись
echo "Привет!" > /proc/telegram/chat_1
echo "Как дела?" > /proc/telegram/chat_1

# 5. Повторное чтение
cat /proc/telegram/chat_1
[User] Привет!
[User] Как дела?

# 6. Очистка
killall telegram_server
sudo rmmod telegram_fs
make clean
```
